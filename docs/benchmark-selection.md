# 基准测试选型报告

**日期：** 2026-06-13  
**项目：** AI 语音绘画助手  
**目标：** 评估替代 LLM 模型在该项目中的表现，确定最优模型选型

---

## 1. 背景

项目当前使用 **DeepSeek V4 Flash** 将自然语言语音指令转换为结构化 JSON 绘图指令，此前记录的成功率约 60%。为确认瓶颈是否在模型能力上，特进行本次多模型基准测试。

测试覆盖 **4 个模型**（含基线），从**准确率**和**延迟**两个维度进行 16 条标准用例的量化评估。

---

## 2. 测试方法

### 2.1 测试用例（16 条，6 大类）

| 类别 | 用例数 | 权重 | 示例 |
|------|:------:|:----:|------|
| 几何图形 | 5 | 1.0 | 画红色圆、画蓝色矩形、画绿色线 |
| 颜色映射 | 2 | 1.2 | 画棕色圆、画青色线 |
| ASR 容错 | 2 | 1.2 | 发红色圆（同音）、画正方型（字形） |
| 复合指令 | 2 | 1.5 | 画红色圆和蓝色矩形 |
| SVG 复杂物体 | 3 | 1.5 | 画一只小狗、画房子、画玫瑰花 |
| 画布控制 | 1 | 1.0 | 清空 |
| 空间推理 | 1 | 1.0 | 画红色圆在左上角 |

### 2.2 评分维度（每用例满分 10 分）

JSON 有效(1) + 状态正确(1) + 动作类型(2) + 参数精确(2) + 最少动作数(1) + 有中文摘要(1) + SVG 有效(1) + 中文摘要(1)

### 2.3 测试环境

- 框架直接调用各 Provider API（不经过 Express 代理），排除网络中间层干扰
- 请求间隔 1-1.5s 防限流
- 同一系统提示词不修改，隔离模型本身能力差异

## 5. 分析

### 5.1 准确率分析

四模型准确率完全一致（95.5%），共享相同的失分点：
1. **复合指令位置**（87.5%）— "画红色圆在左边和蓝色矩形在右边" 有时位置对调
2. **SVG 有效性**（88.9%）— 部分模型返回空 SVG 或用 draw_shape 替代
3. **清空指令**（85.7%）— 部分模型返回 success 但没有 canvas_control 动作

这证明**在系统提示词固定的条件下，模型间没有精度差异**。进一步的精度提升需要通过提示词优化或结构化输出约束（如 Qwen 的 `response_format: { type: "json_object" }`）实现，而不是换模型。

### 5.2 延迟分析

延迟是四模型间的**唯一显著差异**：

| 模型 | 延迟特征 | 语音交互适用性 |
|------|---------|:------------:|
| DeepSeek V4 Flash | 亚 500ms，无异常峰值 | ✅ 完美 |
| DeepSeek V4 Pro | 亚 500ms，比 Flash 快 14% | ✅ 完美 |
| Qwen3.7-Max | 2-9s，推理模型固有延迟 | ⚠️ 可用但体验差 |
| Kimi K2.6 | 5-56s，SVG 超过 30s | ❌ 不可用 |

---

## 6. 结论

### 最终推荐：DeepSeek V4 Flash（保持现状）

**理由：**
1. **准确率无差异** — 四模型 95.5% 完全相同，换模型不提升精度
2. **延迟最优** — 265ms 平均延迟，远低于 Qwen（3.6s）和 Kimi（14s）
3. **成本最低** — 已有 API Key，无需额外付费
4. **架构最简** — 无需为多 Provider 增加代码复杂度

> 如果追求极致延迟，可切换至 **DeepSeek V4 Pro**（169ms，使用同一 API Key），但 265ms 对语音交互已足够。

### 后续方向

精度提升的路径不在换模型，而在于：
1. **提示词工程** — 优化 JSON 格式示例，增加失败示范（few-shot）
2. **结构化输出** — 利用 Qwen 等模型的 `response_format` 参数强制 JSON 输出
3. **后处理回退** — 前端对 LLM 返回做二次校验和修正
