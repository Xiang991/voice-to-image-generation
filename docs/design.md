# AI 语音绘图助手 — 设计文档

## 项目概述

纯语音控制的 AI 绘画工具。用户通过语音指令完成绘图创作。

**核心机制：** 语音识别 → 厚前端解析 → 薄代理转发 → LLM 以"绘画设计师"角色输出 JSON → 前端状态判断 → Canvas 渲染

## 总体架构

```
用户语音 → 厚前端（浏览器）
  ├── ASR 转文本（Web Speech API）
  ├── 调用代理层（agent.js）
  ├── 解析 LLM 返回的 { status, actions[], summary }
  ├── 状态判断：success / optimized / error
  ├── Canvas 2D 渲染（Canvas.jsx）
  └── 更新画布摘要（canvasSummary.js）

厚前端 → 薄代理（Express）
  ├── 接收 { text, canvasSummary }
  ├── 注入 API Key + 系统提示词
  ├── 坐标校验 validateActions()
  └── 透传 LLM 响应

薄代理 → 云 LLM（DeepSeek V4 Flash）
  └── 以"绘画设计师"角色理解自然语言 → 绘图指令
```

### 架构原则

| 层级 | 职责 | 决策权 |
|------|------|--------|
| 前端（厚） | ASR / 调代理 / 解析 JSON / 状态判断 / 渲染 / 存储 | 全部业务判断 |
| 代理（薄） | 接收请求 / 加 Key / 坐标校验 / 转发 | 仅坐标校验 |
| LLM（云） | 理解意图 / 规划构图 / 设计画面 / 输出指令 | 设计决策 |

## 核心设计：LLM 作为"绘画设计师"

### 旧模式的问题

之前 LLM 被定位为"绘图助手"，本质是一个**格式转换器**：接收中文文本，输出 JSON。系统提示词满是约束条件（颜色映射、坐标范围、尺寸约定），但从未要求 LLM 思考画面、规划构图、追求美感。结果就是输出合规但画面像"儿童简笔画"。

### 新模式：四步设计流程

重写的系统提示词将 LLM 从被动执行者升级为主动创造者：

**角色定位：** 你是一位经验丰富的**绘画设计师和空间规划师**。

每次绘制前，LLM 在大脑中完成：

```
第一步：逐元素解析用户意图
  ├── 提取每个元素的颜色/形状/位置
  ├── 强制检查颜色-形状绑定（防止互换）
  └── 同音字纠错 + 模糊指令创造性填充

第二步：构图规划
  ├── 主次分明，确定视觉重心
  ├── 多元素协调形成画面而非散落零件
  └── 利用画布空间，饱满不拥挤

第三步：确定表达方式
  ├── 几何（draw_shape）→ 基础形体
  └── SVG（draw_svg）→ 复杂物体和场景

第四步：细化参数
  ├── 精确坐标、尺寸、颜色
  ├── 检查是否在画布范围内
  └── 检查重叠和比例协调
```

### SVG 艺术质量标准（六大技法）

提升 SVG 从"简笔画"到"绘画级"质感的关键技法：

| # | 技法 | 要求 | 示例 |
|---|------|------|------|
| 1 | **渐变** | 每个主要物体使用 linear/radialGradient | 天空渐变、球体径向光晕 |
| 2 | **贝塞尔曲线** | 有机形体用 C/Q 曲线而非几何堆砌 | 动物轮廓、植物枝叶 |
| 3 | **分层构造** | 背景→中景→前景，opacity 创造景深 | 远景山→中景树→前景花 |
| 4 | **光影立体感** | 半透明投影 + 高光 | 花瓣高光、物体投影 |
| 5 | **丰富配色** | 十六进制色码，每物体 ≥ 3 色调 | 基色+阴影+高光 |
| 6 | **视口规范** | viewBox="0 0 800 600" | 与画布一致 |

**few-shot 示例：** 提示词中包含一个完整的日落风景 SVG（带渐变天空、太阳光晕、贝塞尔曲线山脉、剪影树），作为高质量参考。

## 数据流

```
用户说 "画一个田园风景"
  │
  ▼ ① VoiceController (Web Speech API → 文本)
  │
  ▼ ② agent.js → fetch POST /api/agent
  { text: "画一个田园风景", canvasSummary: [] }
  │
  ▼ ③ Express 代理
  ├── 注入系统提示词（绘画设计师角色）
  ├── 构造 messages: [{role:system}, {role:user}]
  └── POST DeepSeek API
  │
  ▼ ④ DeepSeek 以设计师角色思考后返回 JSON
  { "status":"success",
    "actions":[{ "type":"draw_svg",
      "params":{
        "svg":"<svg viewBox='0 0 800 600'>...<defs><linearGradient.../></defs>...",
        "x":0,"y":0,"scale":1 }
    }],
    "summary":"画了一幅风景画：蓝天渐变，草地起伏，一条蜿蜒的河流穿过..." }
  │
  ▼ ⑤ 代理透传 → 前端解析
  │
  ▼ ⑥ App.jsx: status === "success"
  ├── Canvas.jsx: 渲染 SVG（含渐变/曲线/分层）
  ├── canvasSummary.js: 更新画布摘要
  └── History.jsx: 添加历史记录
```

### 数据流对比

| 阶段 | 旧模式 | 新模式 |
|------|--------|--------|
| LLM 角色 | 被动"绘图助手" | 主动"绘画设计师" |
| 输出 SVG 长度 | ~600 字符 | ~2500+ 字符 |
| SVG 技法 | 纯色填充、几何堆砌 | 渐变、贝塞尔曲线、分层、光影 |
| 场景感 | 零散元素 | 主题统一、有凝聚力的画面 |
| 颜色 | CSS 颜色名 | 十六进制色码 + 多色调 |

## LLM 输出格式

```json
{
  "status": "success" | "optimized" | "error",
  "actions": [
    {
      "type": "draw_shape" | "draw_svg" | "canvas_control",
      "params": { ... }
    }
  ],
  "summary": "中文描述设计了什么"
}
```

### status 取值与前端行为

| 值 | 含义 | 前端行为 |
|----|------|---------|
| `success` | 成功解析并生成完整指令 | 执行 actions，显示 summary |
| `optimized` | 优化了模糊指令 | 执行 actions，显示 "已优化："+summary |
| `error` | 无法理解指令 | 不执行，提示用户重说 |

### actions 支持的工具

| 工具 | 说明 | 参数 |
|------|------|------|
| `draw_shape` | 几何图形 | shape, color, x, y, radius/width/height |
| `draw_svg` | SVG 自由图形 | svg 字符串, x, y, scale |
| `canvas_control` | 画布管理 | action: "clear" / "undo" |

## 指令能力

| 类别 | 示例 | 处理方式 |
|------|------|---------|
| 几何图形 | "画一个红色的圆" | draw_shape |
| 自由物体 | "画一只小狗" / "画玫瑰花" | draw_svg（含渐变/曲线） |
| 场景构图 | "画一个田园风景" | draw_svg（完整场景，多层构建） |
| 复合指令 | "画红圆和蓝方块" | actions 数组 |
| 位置指令 | "圆在左边矩形在右边" | spatial |
| 画布控制 | "清空" / "撤销" | canvas_control |
| ASR 容错 | "发房子"（同音字） | 系统提示词纠正 |

## 技术选型

| 模块 | 选型 | 理由 |
|------|------|------|
| 前端框架 | React 19 + Vite 8 | 开发体验好，HMR 快 |
| 语音识别 | Web Speech API | 零成本，浏览器内置 |
| 画布渲染 | 原生 Canvas 2D API | 轻量，SVG 通过 Image 渲染 |
| Agent LLM | DeepSeek V4 Flash | 265ms 延迟，95.5% 准确率 |
| 代理层 | Node Express | 轻量、成熟 |

## 容错策略

| 场景 | 处理方式 |
|------|---------|
| ASR 同音错字 | 系统提示词指明常见错误映射（发→画、园→圆） |
| 模糊指令 | LLM 返回 status: "optimized"，自主决定最优参数 |
| 无法理解 | LLM 返回 status: "error"，前端提示用户重说 |
| 代理/网络错误 | 前端 catch 异常，提示"出错了，请重试" |
| 坐标越界 | 代理层 validateActions() 钳制到画布范围内 |
| SVG 加载失败 | Canvas 显示占位框，不崩溃 |

## 基准测试

内置 16 用例 × 10 维度评分框架。见 `benchmark-selection.md` 详细报告。

**关键结论：**
- 四模型 95.5% 同分——瓶颈不在模型，在提示词表达
- DeepSeek V4 Flash 以 265ms 为最优选型
- 精度提升路径：提示词工程 > 模型换型
